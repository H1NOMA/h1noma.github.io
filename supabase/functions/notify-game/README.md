# Push-уведомления о новых играх — развёртывание

Клиент (сайт) уже всё умеет: кнопка «Уведомления о новых играх» в аккаунте подписывает
устройство, а галочка «Оповестить» при создании игры дёргает эту функцию. Осталось один
раз задеплоить саму функцию — она и рассылает пуши.

## 1. Установить Supabase CLI (если ещё нет)
```
npm i -g supabase
supabase login
supabase link --project-ref xstrdpoxwbkbumigspdv
```

## 2. Задать секреты (VAPID-ключи)
Публичный ключ уже вшит в сайт (`VAPID_PUBLIC` в index.html). Приватный — только здесь,
никуда больше не выкладывать.
```
supabase secrets set \
  VAPID_PUBLIC="BJjgF7Dfj-PGEBho4AK_eec8YivMgnT5Oux8KdfLOx3sDcqpEBJ22tYzdsQ0fhx3S5IlRr-y3zTqsnOszzEpFI4" \
  VAPID_PRIVATE="<ПРИВАТНЫЙ_КЛЮЧ_ИЗ_ЧАТА>" \
  VAPID_SUBJECT="mailto:admin@komikdnd.ru"
```
(SUPABASE_URL и SUPABASE_SERVICE_ROLE_KEY платформа подставит сама.)

## 3. Задеплоить функцию
```
supabase functions deploy notify-game
```

Готово. Теперь при создании игры с галочкой «Оповестить» всем подписавшимся приходит пуш.

## Проверка
1. На телефоне: добавь сайт на главный экран (iOS 16.4+ — обязательно, иначе push не работает),
   открой из иконки, в аккаунте нажми «Включить уведомления», разреши.
2. С аккаунта hinoma/herr_teo создай игру с галочкой «Оповестить» — должно прийти уведомление.

## Заметки
- Рассылать может только аккаунт из списка `DEV` в index.ts (hinoma, herr_teo).
- Мёртвые подписки (410/404) функция сама вычищает из `kv → push:subs`.
- Обновить VAPID-ключи потом = поменять и в index.html (`VAPID_PUBLIC`), и в секретах, и
  переподписать устройства.
